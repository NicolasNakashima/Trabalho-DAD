O que é Docker-Compose

Parabéns por chegar ao 4° assunto do GoDocker! Agora iremos explorar o assunto de Docker-Compose.

Docker compose é uma ferramenta para definir e executar aplicativos Docker de vários contêineres, com o Compose, você usa um arquivo YAML para configurar os serviços do seu aplicativo. O Compose funciona em todos os ambientes: DEV, QA e PROD. Com um único comando, você cria e inicia todos os serviços de sua configuração.

Neste exemplo acima, temos um script de Docker Compose que está configurando dois containers: Container Web e Container Redis. Este script também define a variável de ambiente e a porta de acesso.

Então, explicando o exemplo passo a passo:

  • Linha 1:

    Esta linha define o nome do arquivo Docker Compose. No exemplo, o nome do arquivo é docker-compose.yml.

  • Linha 2:

    Define a versão do arquivo Docker Compose (no exemplo, a versão 3).

  • Linha 3:

    Marca o início da seção de serviços do arquivo.

  • Linha 4:

    Define um serviço com o nome especificado (no exemplo, web).

  • Linha 5:

    Indica como construir a imagem do container (no exemplo, a partir do diretório atual).

  • Linha 6/7:

    Define o mapeamento de portas (no exemplo, expõe a porta 5000 do host para a porta 5000 do container).

  • Linha 8/9:

    Define um volume para o container (no exemplo, mapeia o diretório atual para o diretório /code dentro do container).

    • Linha 10/11:

    Define uma variável de ambiente para o container (no exemplo, define FLASK_ENV como development).

  • Linha 12:

    Define outro serviço com o nome especificado (no exemplo, redis).

  • Linha 13:

    Define a imagem a ser utilizada para o container (no exemplo, a imagem oficial redis:alpine).
